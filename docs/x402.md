# x402: machine-payable claim checking

Groundcheck's hosted engine can charge AI agents per call in USDC over the
[x402 protocol](https://x402.org) — HTTP 402 plus a signed stablecoin transfer
authorization, verified and settled through a facilitator. No accounts, no API
keys, no card rails. Local engines are unaffected: run your own, it is free.

## What is paid, what is free

| Surface | Price |
|---|---|
| `POST /verify` (single claim) | free forever, rate-limited |
| `GET /search`, `/health`, landing demo | free forever |
| `POST /check` (batch document check) | free daily quota per IP, then **$0.02/call** |

`/verify` is the adoption funnel; `/check` re-runs retrieval + stance
classification per claim across a whole document, which is the surface that
actually burns budget.

## Enabling it (operator)

Dormant by default. It exists only when a receiving address is set:

```bash
export GROUNDCHECK_X402_PAY_TO=0xYourAddress        # e.g. a Ledger-held account
```

All dials:

| Env | Default | Meaning |
|---|---|---|
| `GROUNDCHECK_X402_PAY_TO` | *(empty = off)* | receiving address |
| `GROUNDCHECK_X402_NETWORK` | `base` | `base`, `base-sepolia`, `polygon`, `arbitrum`, `ethereum`, or a CAIP-2 id |
| `GROUNDCHECK_X402_FACILITATOR` | `https://x402.org/facilitator` | verify/settle service (default is **testnet-only**) |
| `GROUNDCHECK_X402_FACILITATOR_AUTH` | *(empty)* | `Authorization` header value, for facilitators that need one |
| `GROUNDCHECK_X402_ASSET` | USDC for the network | ERC-20 contract address |
| `GROUNDCHECK_X402_PRICE_CHECK` | `0.02` | USD per `/check` call |
| `GROUNDCHECK_X402_FREE_PER_DAY` | `5` | free `/check` calls per IP per UTC day (`0` = none) |

### Testnet first (recommended)

The default facilitator (`x402.org`) settles **Base Sepolia only** — perfect
for a dry run with test USDC and zero real money:

```bash
export GROUNDCHECK_X402_PAY_TO=0x...      # any address you control
export GROUNDCHECK_X402_NETWORK=base-sepolia
```

### Mainnet (real USDC)

Point the facilitator dial at one that settles Base mainnet — e.g. Coinbase's
CDP facilitator (1,000 free settlements/month, then $0.001 each, KYT screening
included; requires CDP API credentials) or any other x402 facilitator:

```bash
export GROUNDCHECK_X402_NETWORK=base
export GROUNDCHECK_X402_FACILITATOR=https://<your-facilitator>
export GROUNDCHECK_X402_FACILITATOR_AUTH="Bearer <token>"   # if required
```

Settling through the CDP facilitator also gets the endpoint indexed in the
**x402 Bazaar** (Coinbase's discovery layer), where agents search for payable
services — no separate listing step needed.

### Self-custody note

`PAY_TO` is just an EVM address. Generate it on a hardware wallet (Base is a
standard EVM chain) and every settlement lands in an address whose keys never
touch a server.

## Protocol support

Both x402 generations are served simultaneously, because the ecosystem is
mid-migration from v1 to v2:

- **A 402 response** carries the v1 JSON body (`{"x402Version": 1, "accepts":
  [...]}`) **and** a v2 `PAYMENT-REQUIRED` header (base64 JSON, CAIP-2
  network ids first).
- **Payment** is accepted from either `X-PAYMENT` (v1) or `PAYMENT-SIGNATURE`
  (v2). The `accepts` array always offers both network spellings (`base` and
  `eip155:8453`), and the facilitator is addressed in the payer's dialect.
- **The receipt** goes out as both `X-PAYMENT-RESPONSE` and
  `PAYMENT-RESPONSE`.

Scheme: `exact` (EIP-3009 `transferWithAuthorization`; gasless for the payer,
gas sponsored by the facilitator).

## Discovery

- `GET /.well-known/x402` — machine-readable manifest: prices, networks,
  accepted versions, the free-quota policy, and which endpoints stay free.
  404s when the feature is off.
- `GET /health` — includes `"x402": true|false`.
- `llms.txt` documents pricing for agents that read it.

## Guarantees (fail-closed by construction)

1. **Off means off** — without `PAY_TO`, behavior is byte-identical to before.
2. **No result on an unverified or unsettled payment** — decode, verify, and
   settle failures all return 402; the computed result is discarded if
   settlement fails.
3. **No charge on a failed result** — settlement only runs after the engine
   returns 200. A crash costs the payer nothing.
4. **Facilitator outage = 402**, never a free ride and never a silent charge.
5. **Paid calls skip the free-surface rate limit**; free-quota accounting
   never counts a paid call.

The full gating logic is covered in `engine/tests/test_x402.py`.

## Try it

```bash
# see the offer
curl -s -X POST https://<host>/check -H 'content-type: application/json' \
  -d '{"text": "The Eiffel Tower was completed in 1889."}' | jq
# -> 402 with accepts[] once the free quota is spent

# machine-readable manifest
curl -s https://<host>/.well-known/x402 | jq
```

Agent-side, any x402 client SDK (e.g. the `x402` npm/pip packages, or an
MCP-native wallet) handles the 402 → sign → retry loop automatically.
